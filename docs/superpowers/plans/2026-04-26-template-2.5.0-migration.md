# Template 2.5.0 Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Align this plugin's Gradle build, properties, and CI configuration with JetBrains/intellij-platform-plugin-template release 2.5.0, while preserving the project's `pluginSinceBuild` / `pluginUntilBuild` values and removing template-bundled features the project does not use (Qodana, Kover/Codecov, opentest4j).

**Architecture:** Move repository configuration and plugin version declarations into `settings.gradle.kts` via the `org.jetbrains.intellij.platform.settings` plugin. Inline what the template inlines (`platformVersion`, `junit`). Delete `libs.versions.toml`, `qodana.yml`, `codecov.yml`. Strip `gradle.properties` / `build.gradle.kts` of redundant or removed-feature configuration. The deliberate deviation from template: keep `pluginSinceBuild` / `pluginUntilBuild` and the `intellijPlatform.pluginConfiguration.ideaVersion {}` block intact, since their values must continue to ship in the built `META-INF/plugin.xml`.

**Tech Stack:** Gradle Kotlin DSL, IntelliJ Platform Gradle Plugin 2.14.0, Kotlin JVM 2.3.21, Gradle Changelog Plugin 2.5.0, Gradle 9.4.1, GitHub Actions.
