# Migration to JetBrains/intellij-platform-plugin-template 2.5.0

## Goal

Align the project's Gradle build, properties, and CI configuration with the
upstream IntelliJ Platform Plugin Template release 2.5.0, while preserving
features the project actively uses (`pluginSinceBuild` / `pluginUntilBuild`
control) and removing template-bundled features it does not use.

## Context

The previous alignment landed in commit `17c4bdb` (template 2.4.1). Template
2.5.0 (release 2026-04-17) is a substantial simplification pass:

- Repository configuration moves to `settings.gradle.kts` via the
  `org.jetbrains.intellij.platform.settings` plugin.
- `libs.versions.toml` is removed; plugin versions are inlined into
  `settings.gradle.kts` and `build.gradle.kts`.
- `pluginGroup`, `pluginVersion`, `pluginName`, `platformVersion`,
  `gradleVersion`, `pluginSinceBuild` properties are removed.
- Qodana, Kover, opentest4j, signing, publishing.token, publishing.channels,
  pluginVerification, jvmToolchain, the `java` plugin id, and the
  `runIdeForUiTests` task are removed from `build.gradle.kts`.

This project actively uses `pluginSinceBuild` / `pluginUntilBuild` and has
explicit values that must continue to ship in the built plugin manifest. Qodana,
Kover, and opentest4j are not used here.

## Decisions

| Area | Decision |
| --- | --- |
| Repositories in `settings.gradle.kts` | Apply (headline 2.5.0 change) |
| `pluginGroup` → `group`, `pluginVersion` → `version` | Apply |
| Drop `pluginName` property | Apply (`plugin.xml` already declares `<name>`) |
| Inline `platformVersion` (`2025.3`) | Apply |
| Drop `gradleVersion` property + `wrapper` task | Apply |
| Drop `libs.versions.toml`, inline plugin versions | Apply |
| Drop `java` plugin, `kotlin { jvmToolchain(21) }` | Apply |
| Drop `intellijPlatform.signing` / `publishing.token` / `publishing.channels` / `pluginVerification` | Apply |
| Drop `runIdeForUiTests` task | Apply |
| Drop `opentest4j` test dep | Apply (redundant since IntelliJ Platform 251+; project targets 2025.3 = 253) |
| Drop Qodana (plugin, `qodana.yml`, `inspectCode` workflow job) | Apply |
| Drop Kover + Codecov (plugin, `kover {}` block, `codecov.yml`, codecov upload step) | Apply |
| **Keep** `pluginSinceBuild` / `pluginUntilBuild` properties + `ideaVersion {}` block | Deviate from template — values are functionally important and must remain in the built `plugin.xml` |

## File-by-file changes

### `settings.gradle.kts`

Adopt the 2.5.0 template structure: declare plugin versions in
`pluginManagement`, apply `org.jetbrains.intellij.platform.settings`, and move
the `repositories` block here under `dependencyResolutionManagement`. Preserve
`rootProject.name = "ij-per-document-history"`.

Plugin versions to declare:

- `org.jetbrains.kotlin.jvm` 2.3.21
- `org.jetbrains.changelog` 2.5.0
- `org.jetbrains.intellij.platform.settings` 2.14.0
- `org.gradle.toolchains.foojay-resolver-convention` 1.0.0 (already present)

### `build.gradle.kts`

- `plugins { ... }` reduces to `kotlin.jvm`, `intellijPlatform`, `changelog`
  (no version declarations, no aliases, no `java` id, no `kover`, no `qodana`).
- Remove `group`/`version` lines (read from `gradle.properties` natively).
- Remove `kotlin { jvmToolchain(21) }`.
- Remove the top-level `repositories { ... }` block (moved to settings).
- `dependencies { intellijPlatform { ... } }`:
  - `intellijIdea("2025.3")` (inlined).
  - `testFramework(TestFrameworkType.Platform)` unchanged.
  - Remove `bundledPlugins`, `plugins`, `bundledModules` (all empty in
    `gradle.properties` today).
- `testImplementation("junit:junit:4.13.2")` (inlined). Remove `opentest4j`.
- `intellijPlatform { pluginConfiguration { ... } }`:
  - Remove `name = providers.gradleProperty("pluginName")` (`plugin.xml`
    provides the name).
  - Remove `version = providers.gradleProperty("pluginVersion")` (Gradle reads
    `version` natively from `gradle.properties`).
  - Keep `description = ...` README extraction unchanged.
  - Replace `changeNotes`'s `providers.gradleProperty("pluginVersion")` source
    with `version.map { ... }` to match 2.5.0 template (avoids a removed
    property reference).
  - **Keep** `ideaVersion { sinceBuild = ... ; untilBuild = ... }` reading from
    `pluginSinceBuild` / `pluginUntilBuild` (deliberate deviation from template).
- Remove `intellijPlatform.signing { ... }`, `publishing { ... }`, and
  `pluginVerification { ... }` blocks.
- Keep `changelog { ... }` block as-is.
- Remove `kover { ... }` block.
- `tasks { ... }`:
  - Remove `wrapper { gradleVersion = ... }` task block.
  - Keep `publishPlugin { dependsOn(patchChangelog) }`.
- Remove the `intellijPlatformTesting { runIde { register("runIdeForUiTests") ... } }`
  block.

### `gradle.properties`

- `pluginGroup` → `group`.
- `pluginVersion` → `version`.
- Remove `pluginName`.
- Remove `platformVersion` (inlined in `build.gradle.kts`).
- Remove `gradleVersion` (use `./gradlew wrapper --gradle-version=X` instead).
- Remove the empty `platformPlugins`, `platformBundledPlugins`,
  `platformBundledModules` lines.
- **Keep** `pluginSinceBuild = 243` and `pluginUntilBuild = 261.*`.
- Keep `kotlin.stdlib.default.dependency = false`,
  `org.gradle.configuration-cache = true`, `org.gradle.caching = true`.

### `gradle/libs.versions.toml`

Delete the file.

### `qodana.yml`

Delete the file.

### `codecov.yml`

Delete the file.

### `.github/workflows/build.yml`

- Drop the entire `inspectCode` job.
- In the `test` job, drop the "Upload Code Coverage Report" step
  (`codecov/codecov-action`).
- Update `releaseDraft.needs` from `[build, test, inspectCode, verify]` to
  `[build, test, verify]`.
- Update the comment header (currently mentions Qodana inspections) to drop
  the Qodana line.
- **Do not** touch existing GitHub Action version pins
  (`actions/checkout@v6`, `actions/setup-java@v5`,
  `gradle/actions/setup-gradle@v6`, `actions/upload-artifact@v7`,
  `jlumbroso/free-disk-space@v1.3.1`) — they are at or ahead of the 2.5.0
  template and are managed by Dependabot.

### `.github/dependabot.yml`

No edits planned. Caveat: with `libs.versions.toml` removed, Dependabot's
Gradle ecosystem support will need to track plugin versions declared in
`settings.gradle.kts`. The upstream 2.5.0 template ships this same layout, so
this is presumably supported; if the next Dependabot run misses kotlin /
changelog / intellij.platform updates, follow up separately.

## Verification

**Capture baseline before any code changes:**

```sh
./gradlew clean buildPlugin
unzip -p build/distributions/*.zip '*/lib/*.jar' > /tmp/baseline-plugin.zip   # placeholder; see below
# Concretely: locate the patched plugin.xml inside the built artifact and copy
# it to /tmp/baseline-plugin.xml. The packaged plugin.xml lives inside the
# plugin's main jar under META-INF/plugin.xml; extract with:
#   unzip -o build/distributions/*.zip -d /tmp/baseline-dist
#   unzip -p /tmp/baseline-dist/*/lib/<plugin-jar>.jar META-INF/plugin.xml > /tmp/baseline-plugin.xml
```

**After the migration, run:**

```sh
./gradlew clean buildPlugin
./gradlew check
./gradlew verifyPlugin
```

Then extract the post-migration `META-INF/plugin.xml` the same way into
`/tmp/migrated-plugin.xml` and confirm:

```sh
diff /tmp/baseline-plugin.xml /tmp/migrated-plugin.xml
```

The diff **must be empty**. The built plugin's patched `plugin.xml` must be
byte-for-byte identical to the pre-migration baseline — this is the contract
that `pluginSinceBuild` / `pluginUntilBuild` / `<name>` / `<id>` / `<version>`
all continue to ship with their current values.

If the diff is non-empty, the migration has changed observable plugin metadata
and must be reconciled before completion.

## Out of scope

- Bumping `pluginVersion` or any `platformVersion` upgrade.
- Plugin source changes.
- IDE run configurations under `.run/`.
- Release / UI-test workflow files.
- Adding `<idea-version>` to `plugin.xml` directly (the build patches it from
  the `ideaVersion {}` block, which we are deliberately keeping).
